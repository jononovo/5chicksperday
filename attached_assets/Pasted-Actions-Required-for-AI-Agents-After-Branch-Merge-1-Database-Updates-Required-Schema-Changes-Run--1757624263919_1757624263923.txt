Actions Required for AI Agents After Branch Merge
1. Database Updates Required
Schema Changes - Run These Commands:
npm run db:push
If you get data-loss warnings, use:

npm run db:push --force
This will create/update the following tables:

campaigns - Campaign management
sender_profiles - User professional identity
target_customer_profiles - Target audience definitions
user_outreach_preferences - Daily outreach settings
daily_outreach_batches - Batch tracking
daily_outreach_items - Individual outreach items
daily_outreach_jobs - Scheduled job tracking
daily_outreach_job_logs - Execution history
CRM Communications History (if included):
If the CRM communications tracking is part of this merge, also create:

communication_history table - Track all outreach
Additional columns on contacts table for status tracking
Link columns on daily_outreach_items
2. Environment Variables to Configure
Ensure these are set in your environment:

Core Email Services:
SENDGRID_API_KEY - For daily nudge emails
SENDGRID_FROM_EMAIL - Sender email (default: quack@5ducks.ai)
SENDGRID_FROM_NAME - Sender name (default: 5Ducks Daily)
APP_URL - Base URL for email links
Gmail Integration:
GMAIL_CLIENT_ID - For Gmail OAuth
GMAIL_CLIENT_SECRET - Gmail OAuth secret
AI Services:
OPENAI_API_KEY - Email content generation
PERPLEXITY_API_KEY - Company research
Payment Processing:
STRIPE_SECRET_KEY - Stripe API
STRIPE_WEBHOOK_SECRET - Webhook security
STRIPE_PRICE_ID - Subscription price ID
3. Package Dependencies
All necessary packages are already installed based on the package.json. No new installations needed.

4. Workflow Updates
The "Start application" workflow should automatically restart after the merge. It runs:

npm run dev
This starts both the Express backend and Vite frontend on the same port.
